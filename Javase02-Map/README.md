## 💡 核心洞察 (Insight)
* **Map 是“秒级查找”的代名词**：在数组中找东西需要一个个比对（$O(n)$），在 Map 中查找就像查字典，直接通过 Key 定位位置（$O(1)$）。
* **Key 的唯一性**：如果多次 `put` 同一个 Key，后面的 Value 会**覆盖**前面的。这在更新 AI 配置时非常有用。

## 🎙️ 面试高频考点
1. **HashMap 的底层结构**：由“数组 + 链表 + 红黑树”组成。
2. **为什么查找快？**：因为它会对 Key 进行 `hash` 运算，计算出它在数组中的具体位置，而不是一个个去比。
3. **get 失败会怎样？**：如果 Key 不存在，`get()` 会返回 `null`。在 AI 业务中，通常配合 `getOrDefault()` 使用来提供默认设置。

- 把 Map 当成“高级数组”来用

- 忽略了 HashMap 底层依赖 `hashCode + equals`

---

### 我最后记住的 3 句话
- Map 的核心不是存值，而是 **通过 Key 快速定位**
- Key 设计不好，Map 等于白用
- HashMap 是否正确工作，取决于 Key 是否稳定
